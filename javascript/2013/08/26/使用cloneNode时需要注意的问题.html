使用cloneNode时需要注意的问题

2013-08-26

oneNode()是DOM中Node对象的方法，使用cloneNode可以方便的复制DOM节点。cloneNode()接收一个参数includeall。includeall为一个布尔值，true表示被clone的节点的所有子节点也会被clone（既深度clone），false(默认)只会clone原节点。

1、当被clone的节点包含script标签时，clone后script标签是否会再次被执行

内嵌script标签：

1 <div id="box">
2     <script type="text/javascript" >alert(1)</script>  
3 </div>  
4 <script type="text/javascript">  
5     document.body.appendChild(document.getElementById('box').cloneNode(true));  
6 </script>

在所有浏览器中alert都只执行一次

外链script标签：

1 <div id="box">
2     <script type="text/javascript" src='clone.js'></script>  
3 </div>  
4 <script type="text/javascript">  
5     document.body.appendChild(document.getElementById('box').cloneNode(true));  
6 </script>

在非IE浏览器中alert只执行一次

在IE中，只有IE6会执行两次

解决方法：在将clone后的节点加入DOM前，手动删除掉里面的script标签

2.当被clone节点被绑定了事件处理函数时，事件处理函数是否会被一同clone

HTML事件处理绑定：

1 <div id="box" onclick='alert(1)'>点我</div>  
2 <script type="text/javascript">  
3     document.body.appendChild(document.getElementById('box').cloneNode(true));  
4 </script>

在所有浏览器中，click事件均被复制

DOM0级事件处理绑定：

1 <div id="box">点我</div>  
2 <script type="text/javascript">  
3     var box=document.getElementById('box');
4     box.onclick=function(){alert(1)}  
5     document.body.appendChild(box.cloneNode(true));  
6 </script>

在所有浏览器中，点击第一个div会有alert，点击第二个div无反应

DOM2级事件处理绑定：

 1 <div id="box">点我</div>  
 2 <script type="text/javascript">  
 3     var box=document.getElementById('box');
 4     if(box.attachEvent){  
 5         box.attachEvent('onclick',function(){alert(1)},false)  
 6     }else{  
 7         box.addEventListener('click',function(){alert(1)})  
 8     }  
 9     document.body.appendChild(box.cloneNode(true));
10 </script>

在非IE浏览器下 点击第二个div不会执行alert

但是在IE6、7、8中 点击第二个div则会执行alert

在《精通javascript》一书中，作者推荐一种Dean Edwards提出的跨浏览器事件绑定/删除事件解决方案

 1 function addEvent(element, type, handler) {  
 2     // 为每一个事件处理函数赋予一个独立ID
 3     if (!handler.$$guid) handler.$$guid = addEvent.guid++;
 4     // 为元素建立一个事件类型的散列表（元素的所有事件类型都保存在该对象中）
 5     if (!element.events) element.events = {};
 6     // 为每一个元素/事件建立一个事件函数处理的散列表(同一事件类型的不同处理函数保存在该对象中)
 7     var handlers = element.events[type];
 8     if (!handlers) {
 9         handlers = element.events[type] = {};
10         // 存储已有事件处理函数（如果存在一个）
11         if (element["on" + type]) {
12             handlers[0] = element["on" + type]; 
13         }
14     }
15     // 在散列表中存储该事件类型的处理函数
16     handlers[handler.$$guid] = handler;
17     // 注册一个全局处理函数来处理所有函数
18     element["on" + type] = handleEvent;
19 }
20 
21 // 创建独立ID计数器
22 addEvent.guid = 1;                                                
23 
24 function removeEvent(element, type, handler) {
25     // 从散列表中删除事件处理函数
26     if (element.events && element.events[type]) {
27         delete element.events[type][handler.$$guid];
28     }
29 };
30                                                             
31 function handleEvent(event) {
32     // 获取event对象 (IE 中使用全局event对象)
33     event = event || window.event;
34     // 获取对应事件的处理函数散列表
35     var handlers = this.events[event.type];
36     // 依次执行处理函数散列表中的函数
37     for (var i in handlers) {
38         this.$$handleEvent = handlers;
39         handlers[i].apply(this,[event]);
40     }
41 };

为了弥补element['on'+type]无法绑定多个处理函数的缺点，addEvent将所有事件类型存储在element的events对象中，events中的每一个事件类型以同样的形式存储着该类型下所有的处理函数

1 element.events={
2     click:{
3         0:function(){...},
4         1:function(){...},
5         ...
6     },
7     mousedown:{...},
8     ...
9 }

总结：

1.在使用cloneNode()时，最好在插入前将clone出来的节点中的script手动清除掉，以避免脚本可能被重复执行。

2.使用cloneNode()会将通过attachEvent绑定的事件复制到clone出来的节点上，可以通过使用跨浏览器的事件绑定解决方案来统一让绑定的事件不被复制。
